Why coding a webserver as there are exisiting and much better ones?

System Programming, especially TCP/IP Client Server Programming, with C is interesting and pretty cool, While coding a webserver by myself, I learned about security risks and how to avoid them.

This time I started with the basic concept before writing one line of new code to avoid workarounds like in the code of the first server. You can check the documentation.

As soon as I reach version 1.0 I will publish the code on github for one who is interested. All pages and pictures shown here (dwell.goip.de:8888) are provided by my new server, as hardware basis I am using a Raspberry PI 3.

Mandelbrot Fractals

In 2k17 I've chosen as task in POSIX C to realize a Pixel Generator which exchanges data over shared memory and coordinates the synchonisatzion between a generator process and a receiver process with semaphores. As one proposal, the generator should calculate the Mandelbrot set in a loop. The calculation itself, thanks to thousands of examples found in internet, was easy to realize, much easier than the interprocess communication.

The main task was not the Mandelbrot calculation, but I was fascinated and started to work and experiment with different color mappings and spent lot of kwh of electrical power with calculating.

I recoded after handing in my task the whole tool to a standalone, 4 thread application for Linux. I will continue working on that stuff, planning (already started) a nice GTK-3 GUI with support for mouse actions, save as JPG possibility, more and intelligent calculation threads and smooth fancy color mapping. Check out mit github respositority, download the code (only LINUX compatible at the moment), compile and have fun calculating some pictures by yourself.
