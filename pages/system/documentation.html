Some more informations

  • independend log process, forked when server is started
  • own process for every access, forked when requested
  • not valid IP adress, errors with timestamp and missing HOST information leads to closing the connection
  • incoming string from webbrower added to temporary linked list for fast serach
  • METHOD check, if not GET or HEAD answer ALLOWED before closing connection
  • URL check for maximum length, if exceeded error 404 page
  • not existing document error 400 page
  • all other errors from functions (standard C and own) lead to error 500 page
  • existing documents scanned at every access with recursive function and stored in temporary linked list for fast serach
  • supported mime types can be changed in header file before compiling server
  • support tested with: htm / html / txt / png / jpg / gif / mp4
